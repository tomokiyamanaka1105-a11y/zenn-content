仕事終わりに深夜までコードを書いている。

きっかけはGAS（Google Apps Script）だった。業務の手作業を自動化したくて触り始めたら、気づいたら深夜まで没頭していた。楽しくて仕方なかった。

それから約1年。今はAIを使ってアプリを2本公開するところまで来た。売上はまだゼロ。でも「公開まで行けた」という事実は大きい。この記録を残しておく。

## GASから始まった

本業の業務が非効率だった。スプレッドシートの手作業、毎日同じ繰り返し。「自動化できないか」と調べてGASに辿り着いた。

最初はコピペで動かすだけ。でも少しずつ書けるようになって、業務時間が半分になった。

**そこで気づいた。「コードで問題を解決する」という感覚が、純粋に楽しい。**

## 個人開発へ

GASで自動化の面白さを知ってから、もっと大きなものを作りたくなった。AIコーディングツールの登場で「これなら自分でも作れる」と確信した。

使い始めたのが **Claude Code**。コードをほぼ全部書いてもらいながら、自分はロジックと設計に集中する。このスタイルが合っていた。

## 現在：公開済みアプリ2本

### IdeaProof（旧DemandIQ）

AIでビジネスアイデアを検証するWebアプリ。海外ユーザー向けに英語で作った。

**技術スタック:**
- バックエンド：Node.js + Express
- AI：Claude / Gemini / OpenAI の3社を用途で使い分け
- 認証：JWT + Google OAuth
- キャッシュ：Upstash Redis
- 決済：Stripe
- デプロイ：Vercel

### MacCleanAgent

Macのディスク容量を解放するデスクトップアプリ。自分が欲しくて作った。

**技術スタック:**
- Electron + Electron Forge
- React（フロントエンド）
- Node.js + Express（決済バックエンド）
- Stripe + Upstash Redis

## 実際に詰まったこと

きれいな話だけ書いても意味がないので、詰まったことも全部書く。

### 1. Vercelの環境変数地獄

IdeaProofのデプロイ時、設定が必要な環境変数が一気に6個以上あった。

```
ANTHROPIC_API_KEY
JWT_SECRET
STRIPE_SECRET_KEY
STRIPE_WEBHOOK_SECRET
UPSTASH_REDIS_REST_URL
UPSTASH_REDIS_REST_TOKEN
GOOGLE_CLIENT_ID
ALLOWED_ORIGIN
```

1個でも漏れると起動時に`[FATAL]`エラーが出てアプリが落ちる。何度やり直したかわからない。

### 2. Stripe Webhookの二重管理

ローカル開発と本番Vercelで、WebhookのURLが別々になる。ローカルは`stripe listen`コマンドで、本番はStripeダッシュボードに登録。この切り替えを忘れて本番で決済が全部失敗した。

### 3. CORSで本番だけ動かない

`ALLOWED_ORIGIN`の設定をlocalhost のままVercelにデプロイしたら、本番から全リクエストが弾かれた。エラーメッセージが不親切で1時間溶かした。

### 4. Electronのパッケージング

MacCleanAgentをDMGで配布しようとしたら、`forge.config.js`の設定ミスでビルドが通らない。Electronのバージョンと各プラグインのバージョン相性が原因だった。

## 今の状況と次の目標

- IdeaProof：公開済み、ユーザーほぼゼロ、売上ゼロ
- MacCleanAgent：公開済み、同上

正直なところ、集客が全くわかっていない。作るのは楽しいけど、使ってもらうのが難しい。
